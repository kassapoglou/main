Compound Logic

Sometimes a single condition isn't enough. Logical operators allow you to combine multiple Boolean values to create complex security checkpoints.

1. AND (&&)

The && operator requires ALL conditions to be true. If even one is false, the entire statement is false.

if (hasID && isPilot) {
  // Runs ONLY if they have an ID AND are a pilot.
}

2. OR (||)

The || operator requires AT LEAST ONE condition to be true. It's great for fallback overrides.

if (isCaptain || overrideCode) {
  // Runs if they are the captain OR have the code.
}

3. NOT (!)

The ! operator reverses a boolean. True becomes false, false becomes true.

if (!hostileShip) {
  // Runs if the ship is NOT hostile.
}
