1. Arrow Functions (=>)

Arrow functions provide a shorter, cleaner syntax for creating functions. You declare them like variables, followed by parameters in parentheses, an "arrow" =>, and the code block.

// Old Way
function scan(target) { return `Scanning ${target}`; }

// Arrow Function Way
const scan = (target) => { return `Scanning ${target}`; };

2. Implicit Return

If your function is only one line long, you can omit the curly braces {} and the return keyword entirely!

const scan = (target) => `Scanning ${target}`;

3. The Laws of Scope

Variables created outside a function are in the Global Scope (everyone can see them). Variables created inside a function are in the Local Scope (they are destroyed as soon as the function finishes, and the outside world cannot access them).
