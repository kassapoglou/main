The Ship's Databanks

Welcome to the CODE-ARENA, Commander! To fly this starship, the main computer needs to remember important information, like your destination or weapon status. We do this using Variables.

Think of a variable as a digital label you attach to a piece of data. You write a name on the label, and you assign information to it using the = sign.

comms_message = "Hello, Starfleet!"
print(comms_message)
TERMINAL OUTPUT: > Hello, Starfleet!

When the computer reads the first line, it saves the text into its memory under the label comms_message. On the second line, it prints that exact text to the screen.

Shifting Cargo (Reassignment)

Space changes fast! You can update a variable at any time, and the ship's computer will automatically forget the old data and keep track of the new value.

target_planet = "Mars"
print(target_planet)

target_planet = "Jupiter" # The ship changes course!
print(target_planet)
TERMINAL OUTPUT: > Mars > Jupiter

Commander's Naming Protocol

To keep the ship's systems from crashing, all pilots must follow strict rules when naming variables:

  • Use Space-Snakes (Underscores): Spaces are strictly forbidden. If you need two words, connect them with an underscore. (e.g., shield_power works, but shield power will crash the system).
  • Letters First: A variable can contain numbers, but it cannot start with one. (e.g., engine_1 is good, but 1_engine is invalid).
  • Be Descriptive: Give your variables meaningful names. crew_size is much better than just c.

System Glitches (Name Errors)

Even the best commanders make typos! If you accidentally misspell a variable, the computer will throw a NameError. It means you asked the computer to open a databank that doesn't exist.

laser_color = "Red"
print(laser_color) # ERROR! Typo in 'laser'
SYSTEM FAILURE: > NameError: name 'lazer_color' is not defined

If you see a NameError, don't panic! Just check your spelling. Programming languages are incredibly strict about spelling, but they don't care about grammar.
