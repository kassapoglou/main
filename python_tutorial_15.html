Bringing Data Back

Functions process data, but what if another part of your ship needs that resulting data? We cannot just use print(). We must use the return keyword to push the data out of the function.

1. Print vs. Return

Printing is like showing the pilot a picture of coordinates on a screen. Returning is like plugging those coordinates directly into the ship's autopilot.

def calculate_shields(power):
    total = power * 2
    return total # Pushes data OUT of the function

2. Capturing Returned Data

Because a function `return`s a value, you can capture that value by setting a variable equal to the function call.

# The function runs, returns 100, and stores it in 'my_shields'
my_shields = calculate_shields(50)

print(f"Shields locked at {my_shields}!")
> OUTPUT: Shields locked at 100!
