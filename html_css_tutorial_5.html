Block vs. Inline

To build complex interfaces, we need invisible "boxes" to group elements together, and "wrappers" to target specific pieces of text. Understanding **Block** vs **Inline** elements is crucial.

1. The Block Container: <div>

The <div> (Division) is a **Block-level** element. It automatically forces a line break before and after it, taking up the full width available. It acts as an invisible cargo crate to hold other elements.

2. The Inline Wrapper: <span>

The <span> is an **Inline** element. It does *not* create a line break. It is used to wrap a specific word or phrase inside a sentence so you can target it later with CSS (like changing its color).
